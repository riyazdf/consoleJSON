Documentation

Logging JSON

consoleJSON.log(theJSON [, Ruleset])

If the second parameter is not provided, this will use the default setup and themes, logging the JSON with bolded keys and newline delimited key-value pairs. If a ruleset parameter is provided, consoleJSON will use this as the ruleset when displaying the JSON.

consoleJSON.filter(theJSON, key)

A convenience method for filtering. This will create a new filter rule, append it to the default ruleset, and log the key-value pair pertaining to the key we're filtering on, as well as the path to the key.

Rulesets

consoleJSON.Ruleset([theme])

There are several built-in themes that configure the color of JSON primitives. If no parameter is provided, the default theme will be used. Themes, passed in as strings, include:

  • default
  • fire
  • candy
  • ras
  • none

Ruleset.addGlobalRule(Rule)

Adds a rule valid in the scope of this ruleset.

Ruleset.addRule(key, Rule)

Adds a rule valid to a specific key. The key can also be a dot-separated path to a key: "parentkey.childkey.grandchildkey". If the key has a ".", then escape it with a forward slash: "my/.key".

Ruleset.addRuleset(key, Ruleset)

Composes two rulesets together at the level of the key, ovewriting any existing rulesets for the key. The key can also be a dot-separated path to a key: "parentkey.childkey.grandchildkey". If the key has a ".", then escape it with a forward slash: "my/.key".

Ruleset.addTopLevelRule(key, Rule)

Adds a top level rule, which applies to all keys in the JSON. The key can also be a dot-separated path to a key: "parentkey.childkey.grandchildkey". If the key has a ".", then escape it with a forward slash: "my/.key".

Ruleset.removeGlobalRule(Rule)

Removes a rule valid in the scope of this ruleset.

Ruleset.removeRule(key, Rule)

Removes a rule valid to a specific key.

Ruleset.removeRuleset(key)

Removes a ruleset valid to a specific key.

Ruleset.removeTopLevelRule(key, Rule)

Removes a top level rule, which applies to all keys in the JSON.

Ruleset.clone()

Returns a clone of the ruleset.

Rules

consoleJSON.Rule(type, attr, val [, target])

These are the building blocks behind Rulesets. The user can specify CSS-like attributes and values for certain keys.

type

The first parameter is the type of the rule. This can be one of three things:

  • filter - for filtering keys
  • style - to change the color or look of text
  • format - to change the layout

attr/val

The second and third parameters are the specific style or format to be modified, and the value it should take on. Think of this as CSS attributes. There are different attributes for different types:

  • type == "style"
    • highlight - color ("#ff0000"|"rgb(255,0,0)"|"red")
    • font_color - color ("#ff0000"|"rgb(255,0,0)"|"red")
    • font_size - pixel size ("14px")
    • font_family - typeface ("Verdana, Geneva, sans-serif")"
    • font_style - "normal"|"italic"|"oblique"
    • font_weight - "normal"|"bold"|{100,200,...,900}
  • type == "format"
    • collapse - boolean (true to collapse key's value)
    • line_length - integer (number of chars)
    • indent_amt - integer (number of spaces)
  • type == "filter"
    • remove - boolean

target

Targets are the things on which the rule will apply. Targets are only used for "style" types. For all other types, target is set to "unused." Here are the targets that can be used:

  • key
  • val
  • number
  • string
  • boolean
  • null
  • undefined
  • array
  • object
  • all
  • unused

Rules Precedence

The precedence for rules is as follows (highest to lowest):

  • Key Specific Rules
  • Top Level Rules
  • Global Rules on keys/values
  • Global Rules on primitives/arrays/objects
  • Global Rules on all
